# Инструкция по созданию постов в блог

Этот документ содержит руководство для AI-агентов и разработчиков по созданию новых постов в блоге.

## Структура репозитория

```
content/
├── ru/
│   └── blog/
│       └── *.md (или *.md)
└── en/
    └── blog/
        └── *.md (или *.md)
```

- **Русские посты**: `/content/ru/blog/`
- **Английские посты**: `/content/en/blog/`

## Формат файлов

Посты создаются в формате **MDX** (Markdown with JSX) с расширением `.md` или обычный **Markdown** с расширением `.md`.

## Имя файла

Имя файла должно быть:
- **Латинскими буквами**
- **В нижнем регистре (lowercase)**
- **Слова разделены дефисами** (`-`)
- **Отражать суть поста**

### Примеры корректных имён:

✅ `telegram-profile-it-2026.md`
✅ `why-do-we-need-ai-agents.md`
✅ `best-llm-for-ocr-2026.md`
✅ `evolution-ai-hr.md`

❌ `Telegram Profile.md` (пробелы, заглавные буквы)
❌ `телеграм-профиль.md` (кириллица)

## Структура поста

### 1. Frontmatter (метаданные)

Каждый пост начинается с блока метаданных в формате YAML, обрамленного тремя дефисами:

```yaml
---
title: "Название поста"
date: 'YYYY-MM-DD'
description: "Краткое описание поста для превью и SEO"
tags: ['Тег1', 'Тег2', 'Тег3']
authors: ['автор1']
language: 'ru'
---
```

#### Описание полей:

- **title** (обязательно): Заголовок поста. Используйте **двойные кавычки**, особенно если есть двоеточия, апострофы или спецсимволы.
- **date** (обязательно): Дата публикации в формате `YYYY-MM-DD` (например, `2026-02-15`).
- **description** (обязательно): Краткое описание поста (1-2 предложения). Используется для превью и SEO.
- **tags** (обязательно): Массив тегов. Примеры: `['AI', 'HR', 'LLM']`, `['IT', 'Карьера', 'Саморазвитие']`.
- **authors** (обязательно): Массив авторов. Обычно `['slavb18']` или `['Iconicompany Team']`.
- **language** (обязательно): Язык поста. `'ru'` для русского, `'en'` для английского.

### 2. Содержание поста

После frontmatter идет основное содержание в формате Markdown/MDX.

#### Рекомендации по структуре:

1. **Вводный абзац** — краткое введение в тему
2. **Заголовки** — используйте `##` для основных разделов, `###` для подразделов
3. **Форматирование**:
   - **Жирный текст**: `**текст**`
   - *Курсив*: `*текст*`
   - `Код`: `` `код` ``
   - Списки: используйте `1.` для нумерованных и `-` для маркированных
   - Цитаты: `>` в начале строки
4. **Эмодзи** — можно использовать для визуального акцента (⚡, 🔑, ✅, ❌, 👤, 💡)
5. **Ссылки**: `[текст ссылки](URL)`
6. **Блоки кода**:
   ````markdown
   ```python
   код
   ```
   ````

## Процесс создания поста

### Шаг 1: Определите язык и папку
- Русский пост → `/content/ru/blog/`
- Английский пост → `/content/en/blog/`

Выберите описательное имя файла (латиница, lowercase, через дефисы).

- Наличие всех обязательных полей
- Осмысленные теги

## Частые ошибки

❌ **Кириллица в имени файла**
✅ Используйте только латиницу

❌ **Отсутствие языкового поля**
✅ Всегда указывайте `language: 'ru'` или `language: 'en'`

❌ **Неправильный формат даты**
✅ Используйте `YYYY-MM-DD`, например `2026-02-15`

❌ **Теги как строка вместо массива**
```yaml
tags: 'AI, HR'  # ❌ неправильно
tags: ['AI', 'HR']  # ✅ правильно
```

❌ **Отсутствие кавычек в title с двоеточием**
```yaml
title: Заголовок: подзаголовок  # ❌ вызовет ошибку
title: "Заголовок: подзаголовок"  # ✅ правильно
```

❌ **Одинарные кавычки в title или description (ломаются при наличии апострофа)**
```yaml
title: 'It's broken'  # ❌ неправильно — апостроф ломает YAML
title: "It's fine"   # ✅ правильно — используйте двойные кавычки
```

❌ **Строки с эмодзи слипаются в один абзац**

Если несколько строк с эмодзи идут подряд без пустых строк между ними, Markdown объединяет их в один абзац и они отображаются в одну строку.

```markdown
❌ сломанные процессы
❌ хаотичная воронка
❌ ручное управление
```
Отрендерится как: `❌ сломанные процессы ❌ хаотичная воронка ❌ ручное управление`

✅ Добавляйте пустую строку между каждым элементом:

```markdown
❌ сломанные процессы

❌ хаотичная воронка

❌ ручное управление
```
