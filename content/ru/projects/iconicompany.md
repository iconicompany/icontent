# 🚀 Iconicompany

## HR-автопилот для проектной занятости и сборки инженерных команд

**Iconicompany** - это автономная AI-платформа, которая заменяет классический рекрутинг, аутстаффинг и частично системных интеграторов, автоматически формируя и запуская команды под конкретные бизнес-задачи.

Платформа работает по принципу:

```text
Проект → AI → команда → запуск → результат
```

а не:

```text
Вакансия → кандидаты → интервью → команда
```

---

# 💡 Ключевая идея

Бизнесу не нужен разработчик.
Бизнесу нужна **работающая команда, которая решит задачу**.

Iconicompany превращает найм из ручного процесса в **автономный конвейер**:

* без рекрутеров
* без сотен резюме
* без хаотичных интервью

👉 с фокусом на **результат, а не людей**

---

# ⚙️ Как работает HR-автопилот

## 1. Вход: описание задачи

Пользователь просто пишет:

> “Нужно сделать fintech MVP с кредитным конвейером”

---

## 2. AI-декомпозиция проекта

Система автоматически:

* определяет домен (fintech)
* выделяет роли
* оценивает сложность
* строит архитектуру команды

```json
{
"roles": ["tech lead", "backend", "frontend", "qa"],
"complexity": "medium"
}
```

---

## 3. Generative Team Builder (Team-as-Code)

Команда формируется как конфигурация:

```json
{
"team": [
{"role": "Tech Lead", "count": 1},
{"role": "Backend", "count": 2},
{"role": "Frontend", "count": 1},
{"role": "QA", "count": 1}
],
"timeline": "10 weeks"
}
```

👉 Команда становится **объектом системы**, а не результатом рекрутинга

---

## 4. Подбор через AI (векторный поиск + скоринг)

Платформа:

* ищет кандидатов в векторном пространстве
* оценивает их навыки
* учитывает опыт и домен
* проверяет доступность

---

## 5. Team Graph (ключевое отличие)

Вместо поиска людей система анализирует:

* кто работал вместе
* как команды взаимодействуют
* какие комбинации дают результат

👉 формируется **граф команд**, а не база резюме

---

## 6. Team Scoring (оценка команды)

Система считает не “подходит ли кандидат”, а:

```text
TeamScore =
skill_fit
+ domain_experience
+ collaboration_score
+ availability
```

👉 это принципиально другой уровень точности подбора

---

## 7. Instant Team (результат за секунды)

Пользователь сразу получает:

* готовую команду
* стоимость
* сроки
* доступность
* прогноз успеха

```text
Delivery: 10 weeks
Cost: $48k/month
Success probability: 82%
```

👉 вместо 2-4 недель - несколько секунд

---

# 🧠 AI-ядро платформы

## 1. Мультиагентная архитектура

Система - это не одна модель, а сеть агентов:

* Агент анализа задачи
* Агент подбора
* Агент интервью
* Агент оценки
* Агент сборки команды

Они взаимодействуют через протоколы A2A/MCP и динамически делегируют задачи

---

## 2. GRPO и outcome-based learning

Платформа обучается не на “правильных ответах”, а на результате:

```text
интервью → найм → успех проекта → обучение
```

👉 система оптимизирует стратегию подбора, а не ответы

---

## 3. Energy Score (новый подход к мэтчингу)

Вместо cosine similarity используется:

* анализ распределений в латентном пространстве
* оценка “энергии соответствия”

👉 устойчивость к “хрупкости векторов” и сложным доменам

---

## 4. ICONIC (I-Stack) - оценка кандидата

Каждый специалист оценивается по 6 векторным сигналам:

* архитектурное мышление
* код
* логика рассуждений
* семантический матчинг
* интервью
* потенциал роста

👉 итог: **Индекс соответствия (0-100)**

---

# 🧩 Новый уровень: работа с командами

## Team Repository (как GitHub, но для команд)

Платформа хранит:

* команды
* их состав
* историю проектов
* метрики

Команда становится:

```text
объектом с рейтингом, историей и переиспользованием
```

👉 можно “форкать” команды и улучшать их

---

## Team-as-a-Service

Вместо:

* найма людей
* аутстаффинга

появляется модель:

👉 **команда как сервис**

---

# 🌐 Федеративная сеть (следующий этап)

Платформа может масштабироваться как сеть:

* агентства = узлы
* данные остаются локальными
* поиск - глобальный

по аналогии с:

* Git
* ActivityPub

👉 создаётся **глобальный рынок команд**, а не резюме
