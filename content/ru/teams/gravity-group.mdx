## Кейсы

### Кейс 1. AI-ассистент для интернет-магазина

**Задача:**
Товар часто оказывается в резерве, но информация об этом на сайте не обновляется. Покупатели не видят реального наличия, менеджеры перегружены вопросами.

**Решение:**
- Чат-бот на базе AI, работающий только в нерабочее время менеджеров (будни с 7 до 16 по МСК)
- Интеграция с системой учёта остатков — бот информирует о реальном наличии товара
- Масштабирование: кнопка «Подобрать аналог» в карточках отсутствующих товаров с вызовом AI-бота

**Результат:**
- ✅ Снижение нагрузки на менеджеров
- ✅ Покупатели получают актуальную информацию 24/7
- ✅ Потенциал для автоматизации подбора аналогов и консультирования

### Кейс 2. Генерация контента для товаров (отзывы + преимущества)

**Задача:**
Для карточек товаров не хватает уникальных текстов, а отзывов мало. Нужно автоматически формировать блоки «Преимущества и особенности по отзывам пользователей».

**Решение:**
- AI анализирует имеющиеся отзывы, описание товара и общую информацию из интернета
- Автоматически генерируется уникальный текст, повышающий конверсию
- Полностью автоматический процесс для всего каталога

**Результат:**
- ✅ Уникализация контента без участия копирайтера
- ✅ Повышение доверия покупателей
- ✅ Масштабирование на тысячи товаров

### Кейс 4. Автоматический подбор рекомендаций и аксессуаров

**Задача:**
Для каждого товара в каталоге нужно рекомендовать связанные аксессуары. Вручную назначать их для тысяч позиций — невозможно.

**Решение:**
- AI анализирует характеристики товара и автоматически подбирает релевантные аксессуары
- Для бензобура CHAMPION AG 252 система рекомендует подходящие буры, масла, топливные смеси и защитные средства
- Работает для всего каталога без ручного участия

**Результат:**
- ✅ Рост среднего чека за счёт перекрёстных продаж
- ✅ Экономия времени менеджеров и контент-менеджеров
- ✅ Актуальные рекомендации для каждого товара

### Кейс 5. Помощник Проектного Руководителя

**Задача:**
Автоматизировать рутинные задачи проектного руководителя: транскрипция встреч, анализ документов, генерация отчётов, работа с базой знаний проекта. Критичное требование — полная локальность для конфиденциальности данных.

**Решение:**

*Обработка встреч:*
- Транскрипция видео с разделением по спикерам (Whisper + pyannote) — час видео за 2 минуты
- Автораспознавание участников по голосу
- Ускорение загрузки на 80–90%

*База знаний:*
- Векторный поиск (Qdrant + OpenSearch) — память о всех встречах и документах
- Чат с AI, имеющим доступ ко всей информации проекта
- Поддержка любых LLM (OpenAI, Anthropic, локальные)

*Документы:*
- Парсинг PDF, DOCX, сканов (OCR)
- Редактирование в OnlyOffice через браузер
- AI-правка документов по текстовым командам (LangGraph + Office.js)

*Оптимизация:*
- Хранилище сжато на 84% (65 → 10.5 МБ на файл)
- Изоляция данных клиентов (multi-tenant)

**Результат:**
- ✅ Локальный AI-ассистент, экономящий часы ежедневно

### Кейс 6. AI-симулятор для тренировки продажников

**Задача:**
Создать онлайн-тренажёр для отработки навыков холодных и тёплых продаж в безопасной среде, где AI играет роль клиента с настраиваемым поведением.

**Решение:**
- **Real-time симуляция звонков:** AI выступает в роли потенциального клиента (холодный/тёплый лид, уровень возражений, заинтересованность)
- **WebSocket архитектура:** обработка аудиопотоков в реальном времени без задержек
- **STT/TTS:** Yandex Cloud Speech Kit — распознавание речи продавца и синтез голоса AI-клиента
- **Динамическая генерация промптов:** адаптивная сборка контекста — AI ведёт себя естественно и непредсказуемо
- **Геймификация:** задача продавца — убедить AI купить

**Результат:**
- ✅ Безопасная среда для обучения продажников
- ✅ Реалистичные диалоги с разными сценариями
- ✅ Рост эффективности отдела продаж без риска для реальных клиентов
