## Публикация и синк (локальный цикл)

GitHub Action авто-синка **отключён** (`.github/workflows/sync.yml`: триггер `push`
закомментирован, оставлен `workflow_dispatch`). Публикуем и синкаем **локально**, по шагам:

```bash
# 1. коммит + пуш
git add content/ assets/ && git commit -m "blog: ..." && git push

# 2. синк контента на сайт iconicompany.com (только загрузка; без LLM/перевода/рассылки)
#    ПРАВИЛЬНАЯ КОМАНДА (проверено, .com обновляется):
bun scripts/sync-content.ts        # POST https://iconicompany.com/nextapi/sync
#    НЕ путать с scripts/content-sync.ts — он бьёт в SYNC_URL из .env
#    (сейчас там iconicompany.ru) — это ДРУГОЙ хост, .com от него НЕ обновится.
#    Пары секунд после пуша достаточно (GitHub codeload успевает отдать свежий zip).

# 3. проверить, что на сайте новая версия
#    URL: https://iconicompany.com/ru/blog/<slug>   (EN: /en/blog/<slug>)
#    проверять по уникальному маркеру именно новой версии, не по общей фразе.

# 4. анонс в канал о новой/обновлённой статье — вручную через скилл publish-post
#    (живой голос, картинка Gemini 3). НЕ авто-генератор.
```

Правила проверки сайта:
- не крутить длинные polling-циклы с большими паузами — хватает пары секунд;
- при проверке искать строку, которой в старой версии точно не было.

## Реворк контента (убрать воду, добавить экспертизу)

- Источник правды — RU (`content/ru/blog/`); EN — перевод.
- Числа/факты для «экспертности» брать РЕАЛЬНЫЕ из существующих KEEP-постов
  или из исследований `/home/slavb18/work/iconicdocs/HANDOFF.md` (провенанс deep-tech).
- Голос — по скиллу `publish-post`: боль читателя (CTO/HR) — сюжет, технология — доказательство.
  Убирать броетри (эмодзи в начале строк, «не X, а Y» подряд, вопрос в конце).
- Переписывать в ТОМ ЖЕ файле (тот же URL — не плодить SEO-дубли). Дату в frontmatter
  можно поднять на текущую, если статья переопубликовывается как свежая.
- Визуал: диаграммы — `viz:diagram`, картинки — `viz:generate-image` (Gemini 3),
  скиллы в `/home/slavb18/work/claude-viz`. Картинка поста кладётся в `assets/blog/<slug>.png`.
